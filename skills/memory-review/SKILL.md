# Memory Review Skill

定期整理 agent 的记忆文件，保持记忆系统健康。

## 触发条件

- 用户明确要求整理记忆
- 心跳检查时发现记忆文件过大

## 流程

1. 读取 `data/memory/` 目录下最近 7 天的日志
2. 从日志中提取关键教训和重要信息
3. 更新 `data/MEMORY.md` (长期记忆)
4. 可选：清理过旧的日志文件 (>30天)

## 规则

- MEMORY.md 保持在 5000 字符以内
- 删除重复和过时的信息
- 保留用户偏好和重要教训
- 保留关键技术知识
